# 知识沉淀助手

当用户要求记录时，执行以下流程：

## 1. 环境保活
```bash
learnlog status
```
后端由 launchd 持久化运行；若未响应，MCP Server 首次调用会自动拉起。

## 2. 生成六步深度分析 (≥2000字)

**先在对话中完整输出分析内容**（用户可在终端看到），结构如下：

**核心结论** → **领域场景案例** → **第一性原理分析** → **图示分析(Mermaid)** → **代码实现** → **STAR复盘+反问**

## 3. 入库（必须用管道，禁止把长文塞进 argv！）

```bash
# ✅ 正确：先写内容到对话，再管道入库——内容零损失
echo "完整 insight 内容..." | learnlog record "主题" --pipe --energy 5 --aha true --type deep-research

# ❌ 错误：insight 放在命令行参数里——超过 500 字必然截断
learnlog record "主题" "insight全文" ...
```

Diagram 换行用真正换行符，不要用字面量 `\n`：
```bash
learnlog update <id> --diagram $'graph TD\n  A-->B\n  B-->C'
```
或使用含换行的变量。

完成后告知 ID + http://localhost:3000
